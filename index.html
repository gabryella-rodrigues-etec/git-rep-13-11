Meu primeiro envio para o repositório remoto.

Perguntas:

  • O que significa o comando git init?

    • O comando git init cria um novo repositório Git vazio em um diretório existente ou reinicializa um já existente. Ele cria uma subpasta oculta chamada .git que contém todos os arquivos necessários para o rastreamento de versões, como histórico de commits, objetos e outras configurações.

  • Para que serve o comando git push -u origin main?

    • O comando git push permite que você envie os commits de sua branch e repositório Git local para o seu repositório remoto. Para poder fazer um git push para seu repositório remoto, você deve garantir que todas as suas alterações no repositório local sejam confirmadas.

  • Qual é a função do comando git status?

    • Permite que você verifique o status da sua atividade no Giit Bash.

  • Por que é importante usar git commit antes de fazer o push?

    • É importante fazer git commit antes do git push para criar um registro local de suas alterações, salvando o projeto em um ponto específico e permitindo que você organize o histórico de versões antes de sincronizá-lo com o repositório remoto. O git push apenas envia os commits já confirmados localmente, então é um passo fundamental para garantir que o que será compartilhado esteja completo e bem documentado.
